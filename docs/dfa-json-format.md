# DFA JSON形式

## 概要

FA Design Gameでは、DFAをJSON形式で提出します。

正確には、内部的にはJSON形式でやりとりしますが、DFA入力用の専用のフォームを使い簡単に送れるようにします

## 内部の形式の例

```json
{
  "states": ["q0", "q1"],
  "alphabet": ["0", "1"],
  "start": "q0",
  "accept": ["q0"],
  "transitions": {
    "q0": { "0": "q1", "1": "q1" },
    "q1": { "0": "q0", "1": "q0" }
  }
}
```

## 各フィールド

### states
状態の集合です。少なくとも1つ以上の要素を含む必要があります
#### 実装
  - 空の入力ボックスを1つ用意されている。
  - 入力ボックスを増やせる。

### alphabet
入力文字の集合です。問題文の中に与えられ、自動で入力されています。

#### 実装
  - 問題文ごとに自動で設定済み

### start

開始状態です。states に含まれている必要があります。1つだけ必ず設定する必要があります。

#### 実装
  - 空の入力ボックスを1つ作る。
  - 入力ボックスは増やせない

## 自動で作成されている項目
- alphabet
- transitionsの状態の部分

## 未入力だと弾かれる項目
- states
  - 少なくとも1つ以上の集合である必要がある
  - 空の入力ボックスを1つ作る。
  - 入力ボックスを増やせる
- start
  - 1つだけ必ず設定する必要がある

- accept
  - 0個でもいい
  - 入力ボックスを増やせる
- transitions
  - 全てのalphabetに対して、どの状態に遷移するのかの入力ボックスが自動で作られる
