## この手順書について

この手順書は、GitHubでのチーム開発が初めての人が、Issueを選ぶところからPull Requestのマージ、Projectのステータス更新までを一人で進められるようにするためのものです。
読み終えると、次のことができるようになります。

- Issueを選んで自分を担当者に設定する
- 作業用ブランチを作り、変更をコミットして送る
- Pull Requestを作り、レビューを受けてマージする

## 読み方

ページは [目次](README.md) の順に並んでいます。
初めての方は、次の [02 GitとGitHubの基礎](02-concepts.md) で全体像と用語を確認してから、実際の手順に進んでください。

各ページの先頭には「← 目次に戻る」と、前後のページへのリンクがあります。
順に追っても、必要なところだけ拾い読みしても構いません。

## 必要な準備

手順を始める前に、以下を用意してください。

- GitHubのアカウントを作り、このリポジトリにアクセスできること
- ローカルにGitをインストールし、リポジトリをcloneしておくこと
- 初回の開発環境セットアップを済ませていること

cloneやセットアップがまだの人は、先に [開発環境ガイド](../dev-setup.md) を読んでください。
