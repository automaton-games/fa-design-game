# Projectのステータスを動かす

> [← 目次に戻る](README.md) | [前: 08 レビューを受けてマージする](08-review-and-merge.md)

## このページで扱うこと

**GitHub Projects** は、IssueやPRをカードとして並べ、進捗を可視化するボードです。
このページでは、作業の進行に合わせてカードのステータスを動かす手順を説明します。

## なぜステータスを動かすのか

ボードを更新すると、いま誰が何をどの段階まで進めているかがチーム全体に分かります。
Assignした時点、PRを作った時点、マージした時点で、カードを右の列へ動かします。

## ボードを開く

リポジトリの `Projects` タブから、使っているボードを開きます。
自分のIssueのカードを探します。

## カードを動かす

作業の段階に合わせて、カードを該当する列へドラッグします。

> **確認事項**: このリポジトリで使っているステータス列の名前は、本ページ執筆時点で未確認です（GitHub Projectsの読み取りに `read:project` スコープが必要だったため）。
> 列名が確定したら、ここに「Assignしたら〇〇、PRを出したら〇〇、マージしたら〇〇」の対応を追記してください。

## ステータスを動かす目安

一般的な進め方では、カードは次のように動きます。
実際の列名は、ボードの設定に合わせて読み替えてください。

1. Assignしたら、着手中を示す列へ
2. PRを作ったら、レビュー待ちの列へ
3. マージされたら、完了の列へ
