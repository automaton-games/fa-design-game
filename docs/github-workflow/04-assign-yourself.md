# 自分をAssignする

> [← 目次に戻る](README.md) | [前: 03 Issueを選ぶ](03-choose-issue.md) | [次: 05 作業ブランチを作る](05-create-branch.md)

## このノートで扱うこと

**Assign（アサイン）** は、Issueの担当者を自分に設定する操作です。
このノートでは、Assignの意味と手順を説明します。

## なぜAssignするのか

Assignには2つの役割があります。

1. ほかのメンバーに「いま自分が作業している」ことを伝える
2. 同じIssueに複数人が重ねて着手するのを防ぐ

このリポジトリでは、作業を始める人が自分をAssigneeに設定します。
複数人で1つのIssueを進める場合も、代表者をAssigneesに入れます。

## Assignの手順

Issue本文の右上にある `Assignees` を開きます。

<!-- スクリーンショット: Assigneesの設定画面 -->

表示された一覧から自分のアカウントを選びます。
Assignees欄に自分のアイコンが表示されれば完了です。

## Assignできない、迷うとき

自分のアカウントが一覧に出ない場合は、リポジトリへの招待が済んでいない可能性があります。
Issueにコメントして管理者に相談してください。

すでに誰かがAssignしているIssueを手伝いたいときは、その人にコメントで声をかけてからにします。
勝手にAssignを上書きしません。
